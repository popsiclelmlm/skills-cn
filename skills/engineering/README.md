# Engineering

我日常用于代码工作的 skills。

- **[diagnose](./diagnose/SKILL.md)** — 面向疑难 bug 和性能回退的纪律化诊断循环：reproduce → minimise → hypothesise → instrument → fix → regression-test。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 一场 grilling session，会用既有领域模型挑战你的计划、打磨术语，并内联更新 `CONTEXT.md` 和 ADRs。
- **[triage](./triage/SKILL.md)** — 通过 triage roles 的状态机来 triage issues。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 根据 `CONTEXT.md` 中的领域语言和 `docs/adr/` 中的决策，寻找让 codebase 更深入的机会。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 搭建其他 engineering skills 会消费的每仓库配置（issue tracker、triage label vocabulary、domain doc layout）。
- **[tdd](./tdd/SKILL.md)** — 使用 red-green-refactor 循环进行测试驱动开发。一次构建或修复一个 vertical slice。
- **[to-issues](./to-issues/SKILL.md)** — 使用 vertical slices，把任意 plan、spec 或 PRD 拆成可独立领取的 GitHub issues。
- **[to-prd](./to-prd/SKILL.md)** — 把当前对话上下文转成 PRD，并作为 GitHub issue 提交。
- **[zoom-out](./zoom-out/SKILL.md)** — 让 agent zoom out，从整个系统的上下文解释一段不熟悉的代码，或给出更高层视角。
- **[prototype](./prototype/SKILL.md)** — 构建一次性 prototype 来充实设计：可以是面向 state/business-logic 问题的可运行 terminal app，也可以是在同一路由中可切换的多种激进 UI 变体。
